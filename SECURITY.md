# Security Policy

## Supported version

This repository is a public proof of concept. Security fixes are best-effort on the default branch.

## Reporting a vulnerability

Do not open public issues for suspected vulnerabilities.

Please report vulnerabilities privately using one of the following channels:

- GitHub private vulnerability reporting (repository **Security** tab -> **Report a vulnerability**)
- If unavailable, contact the maintainers via your private communication channel

Include:

- affected file/module and behavior
- reproduction steps
- impact assessment
- suggested mitigation (if available)

## Disclosure

After a fix is available, maintainers may publish a short advisory and mitigation notes.
